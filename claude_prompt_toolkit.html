Prompt modifier toolkit

Community-tested conventions — not official Claude features. Effect is real but varies.

OODA
Observe, Orient, Decide, Act
Forces deliberate reasoning before responding. Claude thinks through context, constraints, and tradeoffs instead of jumping to an answer.
Best for: complex strategy, ambiguous decisions, anything where the obvious answer might be wrong
Reasoning Start of prompt
Think step by step
Chain of thought
Tells Claude to show its work. Improves accuracy on math, logic, and multi-step problems by forcing intermediate reasoning to be explicit.
Best for: math, logic puzzles, debugging, structured analysis
Reasoning Anywhere
Before answering, consider...
Pre-flight reasoning
Explicitly names the dimensions Claude should weigh before generating a response. More targeted than OODA for specific problem types.
"Before answering, consider the legal, financial, and reputational angles."
Reasoning Start of prompt
L99
Max expertise level
Signals Claude to respond at the highest domain expertise level. Drops beginner framing, assumes deep knowledge, delivers high-density nuanced output.
Best for: clinical supplement analysis, VC-level strategy, institutional-grade research
Tone / depth End of prompt
ELI5 / ELI15
Explain like I'm 5 / 15
Calibrates explanation complexity. ELI5 = total beginner, no jargon. ELI15 = smart teenager, can handle concepts but needs plain language.
Best for: onboarding, complex topics for non-experts, simplifying dense material
Tone / depth Anywhere
Be direct / No hedging
Confidence mode
Removes unnecessary caveats, qualifiers, and softening language. Claude gives a clear answer instead of padding with "it depends" and "some might say."
Best for: when you need a decisive recommendation, not a balanced overview
Tone / depth Anywhere
You are a [role]
Persona / role assignment
Frames Claude's perspective through a specific expert lens — VC analyst, pharmacist, seasoned coach. Shapes both tone and the knowledge base it draws from.
"You are a seasoned Series B investor reviewing this pitch deck."
Tone / depth Start of prompt
Respond only in [format]
Format constraint
Hard constraints on output structure. JSON, bullet points, numbered list, table, one sentence, etc. Prevents Claude from choosing its own format.
"Respond only in a JSON object with keys: recommendation, rationale, risk_level"
Output format End of prompt
Give me 3 options
Multiple alternatives
Forces Claude to generate more than one answer. Useful when you want range and variety rather than a single "best" answer.
"Give me 3 versions: aggressive, neutral, and conservative"
Output format Anywhere
No fluff / Bottom line first
Brevity constraint
Cuts preamble, summaries, and padding. Claude leads with the answer, not a wind-up. Useful when you already have context and just need the output.
Best for: quick answers, high-volume use, when you're time-constrained
Output format Anywhere
Use XML tags
Structured output via XML
Tells Claude to wrap output sections in XML tags like <analysis> <recommendation> etc. Makes parsing and extracting specific sections easy, especially in automation.
"Return your answer with <summary> and <action_items> tags"
Output format End of prompt
[Artifact] at end of prompt
Artifact context injection
Attach an existing artifact to the end of your message so Claude has direct access to the content — no copy/paste needed. Enables clean iterative editing.
Best for: revising documents, extending code, reformatting existing work
Placement End of prompt
Steel man this
Best-case argument mode
Tells Claude to make the strongest possible case for a position — including ones it disagrees with. Opposite of a straw man. Great for debate prep or stress-testing ideas.
Best for: testing your pitch, anticipating objections, exploring positions you're skeptical of
Reasoning Start of prompt
What am I missing?
Blind spot scan
After getting an answer, asking this surfaces assumptions, risks, and perspectives Claude held back. One of the most underused follow-up prompts.
Best for: strategic decisions, pitches, plans where you may have tunnel vision
Reasoning Follow-up prompt
Assume I know [domain]
Knowledge assumption
Skips foundational explanations for a domain you already know. More targeted than L99 — sets expertise level for a specific field without going max on everything.
"Assume I know adult learning theory and instructional design basics."
